## Connecting an Integration

### Via Dashboard (Recommended)

1. Navigate to **Integrations** in your dashboard
2. Click **Connect** next to the integration you want
3. Authorize the integration in the OAuth popup
4. Wait for the success confirmation

### Via API

```typescript
// Step 1: Get OAuth authorization URL
const { authUrl } = await client.integrations.getAuthUrl({
  provider: "gmail",
  redirectUrl: "https://your-app.com/oauth/callback"
});

// Step 2: Redirect user to authUrl
window.location.href = authUrl;

// Step 3: Handle callback with authorization code
const connection = await client.integrations.completeAuth({
  provider: "gmail",
  code: "authorization_code_from_callback"
});
```

## Managing Connections

### List All Connections

```typescript
const connections = await client.integrations.listConnections();

connections.forEach(conn => {
  console.log(`${conn.provider}: ${conn.status}`);
  console.log(`Connected: ${new Date(conn.connectedAt)}`);
  console.log(`Last sync: ${new Date(conn.lastSyncAt)}`);
});
```

### Disconnect an Integration

```typescript
await client.integrations.disconnect("gmail");
```

### Reconnect (Refresh Token)

```typescript
// If connection expires or has errors
await client.integrations.reconnect("gmail");
```

## Data Sync

### Automatic Sync

TinyBrain automatically syncs data from connected integrations:

- **Gmail**: Every 15 minutes
- **Calendar**: Every 30 minutes
- **Slack**: Every 10 minutes
- **QuickBooks**: Every 1 hour
- **Drive**: Every 30 minutes
- **Outlook**: Every 15 minutes

### Manual Sync

Trigger a sync manually:

```typescript
await client.integrations.sync("gmail");
```

### Sync Status

Check when data was last synced:

```typescript
const connection = await client.integrations.getConnection("gmail");
console.log('Last sync:', new Date(connection.lastSyncAt));
console.log('Next sync:', new Date(connection.nextSyncAt));
console.log('Items synced:', connection.totalItemsSynced);
```

## Scopes & Permissions

Each integration requests specific OAuth scopes:

### Gmail Scopes
```
https://www.googleapis.com/auth/gmail.readonly
https://www.googleapis.com/auth/gmail.metadata
```

### Google Calendar Scopes
```
https://www.googleapis.com/auth/calendar.readonly
https://www.googleapis.com/auth/calendar.events.readonly
```

### Slack Scopes
```
channels:read
channels:history
users:read
im:read
im:history
```

**TinyBrain never requests write permissions by default.** We only read data to create memories.

## Security & Privacy

### Data Handling

- ✅ All data encrypted in transit (TLS 1.3)
- ✅ OAuth tokens encrypted at rest
- ✅ Multi-tenant isolation (your data is isolated from others)
- ✅ Tokens automatically refreshed before expiration
- ✅ Can disconnect and delete all synced data anytime

### What TinyBrain Stores

For each integrated item (email, event, message):
- **Content**: Text content for memory creation
- **Metadata**: Timestamp, sender, participants
- **Embeddings**: AI-generated vectors for search
- **Links**: References to original items (URLs)

**What TinyBrain does NOT store:**
- ❌ Raw OAuth tokens (only encrypted hashes)
- ❌ Passwords
- ❌ Payment information (except QuickBooks metadata)
- ❌ Private files contents (only metadata)

## Troubleshooting

### Connection Failed

**Possible causes:**
- User denied permissions
- OAuth redirect URI mismatch
- Token expired

**Solution:**
```typescript
// Try reconnecting
await client.integrations.reconnect("gmail");
```

### Sync Errors

**Check sync logs:**
```typescript
const logs = await client.integrations.getSyncLogs("gmail", {
  limit: 10
});

logs.forEach(log => {
  if (log.status === 'error') {
    console.error(`Sync failed: ${log.error}`);
  }
});
```

### Duplicate Memories

If you see duplicate memories from integrations:

1. Check deduplication settings
2. Verify sync timestamps
3. Contact support if issue persists
